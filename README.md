# Уведомления в Telegram

Приложение для Android перехватывает уведомления от выбранных приложений и отправляет их в Telegram-канал. Управление устройствами и настройками — через Telegram-бота.

## Состав проекта

- **android-app** — Android-приложение (APK): перехват уведомлений, отправка на сервер.
- **bot-server** — сервер: HTTP API для приёма уведомлений и Telegram-бот для управления.

## Быстрый старт

### 1. Telegram-бот и канал

1. Создайте бота через [@BotFather](https://t.me/BotFather), получите токен.
2. Создайте канал, добавьте бота в канал как администратора (с правом публикации).
3. Узнайте ID канала (например, переслав сообщение из канала боту [@userinfobot] или через API).

### 2. Сервер бота

```bash
cd bot-server
cp .env.example .env
# Отредактируйте .env: TELEGRAM_BOT_TOKEN, TELEGRAM_CHANNEL_ID (или задайте через бота: /channel)
pip install -r requirements.txt
```

Запуск (оба процесса):

```bash
# Вариант 1: API и бот в одном процессе (для теста)
python run_all.py

# Вариант 2: раздельно (рекомендуется)
# Терминал 1 — API на порту 8000
uvicorn api:app --host 0.0.0.0 --port 8000

# Терминал 2 — бот
python run_bot.py
```

Укажите в `.env` или задайте через бота:

- `API_BASE_URL` — публичный URL вашего сервера (например `https://your-domain.com`), чтобы пользователи видели его в боте при добавлении устройства.

### 3. Android-приложение

1. Соберите APK:
   ```bash
   cd android-app
   # Если нет gradlew — откройте проект в Android Studio или выполните: gradle wrapper
   ./gradlew assembleRelease   # Windows: gradlew.bat assembleRelease
   ```
   APK: `android-app/app/build/outputs/apk/release/app-release.apk`

2. Установите приложение на устройство.

3. В Telegram-боте выполните `/new` (при необходимости укажите имя устройства). Скопируйте **токен устройства** и **URL сервера**.

4. В приложении на телефоне:
   - Включите доступ к уведомлениям (кнопка «Включить в настройках»).
   - Введите URL сервера и токен устройства, нажмите «Сохранить».
   - В блоке «Приложения для перехвата» добавьте приложение **Сообщения** для SMS (например `com.google.android.apps.messaging` или `com.android.mms`).
   - В блоке **«От кого принимать»** при необходимости добавьте номера или имена отправителей — тогда будут пересылаться только SMS от них. Пустой список = от всех.

5. Если канал ещё не задан в боте — в боте: `/channel -1001234567890` (ваш ID канала).

После этого уведомления от выбранных приложений будут приходить в канал.

## Управление через бота

Отправьте боту <b>/start</b> — откроется меню с кнопками:

• <b>📱 Устройства</b> — список устройств, просмотр токена и приложений, удаление
• <b>⚙️ Настройки</b> — канал для уведомлений
• <b>📢 Канал</b> — показать/установить канал (или команда /channel -100...)
• <b>➕ Новое устройство</b> — добавить устройство и получить токен

Команды (можно вводить вручную): /new, /devices, /channel, /setname, /setpackages, /delete

## Защита от ошибок

- **Android**: отправка уведомлений в фоне не падает при сбое сети; повторная отправка при следующем уведомлении.
- **Сервер**: неверный или неизвестный токен устройства возвращает 403; при ошибке отправки в Telegram — 502 с логированием.
- **Бот**: проверка существования устройства по токену перед изменением имени/пакетов/удалением.

## Несколько устройств

Каждое устройство регистрируется отдельно через `/new`. У каждого свой токен. В приложении на каждом телефоне указывается один и тот же URL сервера и **свой** токен. Все уведомления по умолчанию идут в один канал; при необходимости можно доработать логику (несколько каналов по устройству).

## Сборка release APK (подпись)

Для установки на устройства нужна подпись. Создайте keystore и укажите в `android-app/keystore.properties` (не коммитьте в репозиторий):

```properties
storePassword=...
keyPassword=...
keyAlias=...
storeFile=path/to/keystore.jks
```

В `app/build.gradle.kts` добавьте блок `signingConfigs` и укажите его в `release`. Подробности см. в [документации Android](https://developer.android.com/studio/publish/app-signing).

## Требования

- Android 7+ (API 24+)
- Python 3.10+
- Доступ к серверу с белым IP или доменом для API и бота

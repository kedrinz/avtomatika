# Развёртывание бота на сервере — пошагово

Сервер: **193.164.149.70**  
Подключение: `ssh root@193.164.149.70` (ввести root-пароль когда попросит).

Порт для API: **8000** (в списке закрытых портов его нет, значит он доступен).

---

## Шаг 1. Подключиться по SSH

На своём компьютере откройте терминал (PowerShell или cmd) и выполните:

```bash
ssh root@193.164.149.70
```

Введите root-пароль от сервера. После входа вы окажетесь в консоли сервера.

---

## Шаг 2. Установить Python и нужные пакеты (если ещё не стоят)

Выполните по очереди:

```bash
apt update
apt install -y python3 python3-pip python3-venv
```

Проверка:

```bash
python3 --version
```

Должна быть версия 3.10 или выше.

---

## Шаг 3. Создать папку для бота и загрузить файлы

**Вариант А — через Git (если проект уже на GitHub):**

```bash
cd /root
git clone https://github.com/kedrinz/avtomatika.git
cd avtomatika/bot-server
```

**Вариант Б — папка уже есть (например `nika`):**

Если на сервере уже есть папка с ботом (как на вашем скриншоте — `nika` с `api.py`, `bot_handlers.py` и т.д.), перейдите в неё:

```bash
cd /root/nika
```

(или куда вы положили файлы бота, например `cd /root/avtomatika/bot-server`).

Убедитесь, что внутри есть файлы: `api.py`, `bot_handlers.py`, `config.py`, `database.py`, `telegram_send.py`, `run_all.py`, `run_bot.py`, `requirements.txt`.

---

## Шаг 4. Создать виртуальное окружение и поставить зависимости

В той папке, где лежит `requirements.txt`, выполните:

```bash
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
```

После этого в начале строки должно появиться `(venv)`.

---

## Шаг 5. Настроить .env

Создайте или отредактируйте файл `.env` в этой же папке:

```bash
nano .env
```

Вставьте (подставьте свой root-пароль если nano спросит сохранение — это пароль сервера):

```env
TELEGRAM_BOT_TOKEN=8778303642:AAE1Lz3-K7SE9HjfFrcwiryzxYV3XvXUkfc
TELEGRAM_CHANNEL_ID=-1003572205719
API_BASE_URL=http://193.164.149.70:8000
```

Сохраните: **Ctrl+O**, Enter, затем выход **Ctrl+X**.

---

## Шаг 6. Проверить запуск (тест)

Не выходя из папки бота и с активным `(venv)`:

```bash
python3 run_all.py
```

Должны появиться строки вроде «API started on http://0.0.0.0:8000» и сообщения от бота. В Telegram напишите боту **/start** — должно прийти меню с кнопками.

Остановить тест: **Ctrl+C**.

---

## Шаг 7. Запустить бота постоянно (через screen)

Чтобы бот и API работали после отключения от SSH:

```bash
screen -S evateam
source venv/bin/activate
python3 run_all.py
```

Отключить экран, но оставить процесс работающим: **Ctrl+A**, затем **D**.

Проверить, что экран запущен:

```bash
screen -ls
```

Должна быть сессия `evateam`.

Вернуться в консоль бота потом:

```bash
screen -r evateam
```

Выйти из screen с остановкой бота: зайти в сессию (`screen -r evateam`) и нажать **Ctrl+C**.

---

## Шаг 8. Открыть порт 8000 в файрволе (если API с телефона не доступен)

Если с телефона не удаётся подключиться к `http://193.164.149.70:8000`, на сервере выполните:

```bash
ufw allow 8000/tcp
ufw reload
```

Если UFW не установлен или не используется — пропустите этот шаг. У хостера порт 8000 обычно уже открыт.

---

## Шаг 9. Настройка приложения EVATEAM на телефоне

1. Откройте приложение **EVATEAM** на телефоне.
2. **Шаг 2. Подключение к серверу:**
   - **URL сервера:** `http://193.164.149.70:8000` (без слэша в конце).
   - **Токен устройства:** получите в Telegram-боте командой **/new** или кнопкой «➕ Новое устройство», скопируйте токен и вставьте в приложение.
3. Нажмите **Сохранить**.
4. Включите доступ к уведомлениям (Шаг 1) и добавьте приложение «Сообщения» для SMS (Шаг 3), при необходимости настройте «От кого принимать» (Шаг 4).

После этого уведомления будут уходить на ваш сервер и публиковаться ботом в канал **-1003572205719**.

## Краткая шпаргалка команд

| Действие | Команда |
|----------|--------|
| Подключиться к серверу | `ssh root@193.164.149.70` |
| Зайти в папку бота | `cd /root/avtomatika/bot-server` или `cd /root/nika` |
| Включить venv | `source venv/bin/activate` |
| Запустить бота (тест) | `python3 run_all.py` |

---

## Если что-то пошло не так

- **Бот не отвечает** — проверьте, что в `.env` верный `TELEGRAM_BOT_TOKEN` и что процесс запущен (`screen -r evateam` и нет ошибок в консоли).
- **Уведомления не приходят в канал** — проверьте `TELEGRAM_CHANNEL_ID` в `.env`, что бот добавлен в канал как администратор с правом публикации.
- **Приложение пишет ошибку при отправке** — проверьте, что на телефоне в настройках указан именно URL `http://193.164.149.70:8000`, что бот запущен на сервере и порт 8000 не закрыт файрволом у хостера.
